## 🚀 Быстрый старт (для новых установок)

### 1. Установка зависимостей
```powershell
cd Волна
pip install -r requirements.txt
```

### 2. Настройка MySQL
```sql
CREATE DATABASE daily_tourism CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3. Выполнение миграций
```powershell
mysql -u root -p daily_tourism < migrations/update_schema.sql
```

### 4. Создание администратора
```powershell
python create_admin.py
# Или с параметрами:
python create_admin.py --username admin --password admin123 --full-name "Администратор"
```

### 5. Проверка системы
```powershell
python check_system.py
```

### 6. Запуск приложения
```powershell
python app.py
```

### 7. Вход в систему
- Откройте: http://localhost:5000
- Логин: `admin`
- Пароль: `admin123` (или тот, что указали в create_admin.py)

## 📖 Основные сценарии использования

### Создание нового пользователя
1. Админ-панель → Пользователи
2. "+ Добавить пользователя"
3. Заполнить: логин, ФИО, пароль, роль
4. Сохранить

### Настройка прав пользователя
1. Админ-панель → Права доступа
2. Выбрать пользователя
3. Настроить:
   - Просмотр всех данных (без ограничений)
   - Экспорт в Excel
   - Подразделения (галочками)
   - Отделы (галочками)
4. Сохранить права

### Просмотр журнала действий
1. Админ-панель → Журнал действий
2. Фильтровать по:
   - Пользователю
   - Действию (вход, создание, обновление, удаление)
3. Просмотр страниц

### Смена пароля
1. Админ-панель → Профиль
2. Текущий пароль
3. Новый пароль (минимум 6 символов)
4. Подтверждение
5. Сменить пароль

## 🛠️ Управление ролями (RBAC)

### Доступные роли

| Роль | Описание | Возможности |
|------|----------|-------------|
| **admin** | Администратор | Полный доступ ко всему |
| **manager** | Менеджер | Просмотр данных + экспорт |
| **user** | Пользователь | Только просмотр своих данных |

### Права на ресурсы

| Ресурс | Действия | Описание |
|--------|----------|----------|
| users | view/create/update/delete | Управление пользователями |
| roles | view/create/update/delete | Управление ролями |
| permissions | view/create/update/delete | Управление правами |
| data | view/export | Работа с данными |
| audit_log | view | Просмотр журнала |
| settings | view/update | Настройки системы |

### Ограничения по данным
- **Подразделения** - пользователь видит только выбранные подразделения
- **Отделы** - пользователь видит только выбранные отделы
- **can_view_all** - игнорировать ограничения (для менеджеров)

## 🔧 Настройка и конфигурация

### MYSQL_CONFIG (в app.py)
```python
MYSQL_CONFIG = {
    'host': 'localhost',
    'port': 3306,
    'user': 'root',
    'password': 'your_password',  # Измените!
    'database': 'daily_tourism',
    'charset': 'utf8mb4',
    'collation': 'utf8mb4_unicode_ci',
    'use_unicode': True,
}
```

### SESSION_TIMEOUT (в app.py)
```python
app.config['SESSION_TIMEOUT'] = timedelta(hours=8)  # Измените при необходимости
```

### SECRET_KEY (ОБЯЗАТЕЛЬНО для production!)
```python
app.config['SECRET_KEY'] = 'your-secret-key-change-in-production'
# Замените на случайную строку!
```

## 📊 API Endpoints

### Аутентификация
- `POST /login` - Вход (form: username, password)
- `GET /logout` - Выход

### Данные
- `GET /api/data` - Данные с учётом прав

### Админка (требует auth)
- `GET /admin` - Главная
- `GET /admin/users` - Список пользователей
- `POST /admin/users` - Создать/обновить пользователя
- `DELETE /admin/users/<id>` - Удалить пользователя
- `GET /admin/permissions/<user_id>` - Права пользователя
- `POST /admin/permissions` - Сохранить права
- `GET /admin/subdivisions` - Подразделения и отделы
- `GET /admin/audit-log` - Журнал (params: page, user_id, action)
- `GET /admin/roles` - Роли
- `GET /admin/stats` - Статистика
- `POST /admin/change-password` - Смена пароля

## 🔒 Безопасность

### Рекомендации для production
1. **Смените SECRET_KEY** на случайную строку
2. **Настройте HTTPS** через nginx
3. **Ограничьте доступ** к БД по IP
4. **Регулярные бэкапы** базы данных
5. **Мониторинг логов** на предмет атак
6. **Сильные пароли** для всех пользователей
7. **Регулярные обновления** зависимостей

### Логирование
Все действия логируются в таблицу `audit_log`:
- Входы/выходы
- Создание/обновление/удаление пользователей
- Изменение прав
- Ошибки аутентификации

## 🐛 Устранение проблем

### "Невозможно подключиться к MySQL"
- Проверьте, что MySQL запущен
- Проверьте пароль в MYSQL_CONFIG
- Убедитесь, что БД 'daily_tourism' существует

### "Таблица audit_log не найдена"
```powershell
mysql -u root -p daily_tourism < migrations/update_schema.sql
```

### "Недостаточно прав"
- Убедитесь, что у пользователя роль 'admin'
- Проверьте права в таблице `permissions`

### "Сессия истекла"
- Увеличьте `SESSION_TIMEOUT` в app.py
- Проверьте время на сервере
